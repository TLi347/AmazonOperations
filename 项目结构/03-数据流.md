## 概览

```
用户操作                 服务端                        数据库
─────────               ──────────────────────         ──────────────
上传 XLSX
  │
  ▼
POST /api/upload   →   识别 fileType
                       parsers[fileType](buffer)
                       │
                       ├─ 产品报表  →  upsert ProductMetricDay  (asin × date)
                       └─ 其他报表  →  upsert ContextFile       (by fileType)
                       │
                       保存原始文件到 context/ 目录
                       │
                       返回 { fileType, snapshotDate, rowCount }
  │
  ▼
Context 面板刷新
页面触发 features API

GET /api/features/*  →  读 CategoryMap（确定 ASIN 范围）
                        读 ContextFile（快照数据）
                        读 ProductMetricDay（时序数据）
                        JOIN / 过滤 / 聚合
                        lib/rules/*(input) → 结果数组
                        返回 JSON
  │
  ▼
Panel 渲染
```

## 一、文件上传流程

### 1.1 前端触发

```
用户在 Context 面板点击「+ 添加文件」
  → <input type="file" accept=".xlsx"> 弹出文件选择框
  → 选择文件后，读取 File 对象
  → 根据文件名推断 fileType（或展示下拉让用户确认）
  → POST /api/upload  multipart/form-data
      { file: File, fileType: string }
```

**fileType 推断规则**（从文件名关键词匹配）：

| 文件名包含 | 推断 fileType |
|-----------|--------------|
| `产品报表` + `ASIN视图` | `product` |
| `搜索词重构` | `search_terms` |
| `广告活动重构` + `ALL` | `campaign_3m` |
| `广告活动` (非重构) | `us_campaign_30d` |
| `广告位` | `placement_us_30d` |
| `库存报表` | `inventory` |
| `关键词监控` | `keyword_monitor` |
| `成本管理` | `cost_mgmt` |
| `ABA` | `aba_search` |

无法推断时展示确认弹窗，用户手动选择。

> **边界情况**：文件名含"广告活动"但既不含"重构"也不含"ALL"（例如 `广告活动汇总.xlsx`），匹配 `us_campaign_30d` 规则前需先排除含"重构"的情况，若仍不确定则弹窗确认。推断规则的匹配顺序：`campaign_3m`（需同时含"重构"和"ALL"）优先于 `us_campaign_30d`（含"广告活动"），避免误判。

### 1.2 服务端处理（`/api/upload/route.ts`）

```ts
export async function POST(req: Request) {
  const formData = await req.formData()
  const file     = formData.get("file") as File
  const fileType = formData.get("fileType") as string

  // 1. 读取文件内容
  const buffer = Buffer.from(await file.arrayBuffer())

  // 2. 调对应 parser
  const parser = parsers[fileType]
  const { rows, snapshotDate } = parser(buffer)
  //  snapshotDate 从报表 D2 单元格提取，格式 "2026-04-08"

  // 3. 写入数据库
  if (fileType === "product") {
    // 产品报表：按 (asin, date) 累积
    await Promise.all(
      rows.map(row =>
        db.productMetricDay.upsert({
          where:  { asin_date: { asin: row.asin, date: snapshotDate } },
          update: { metrics: row },
          create: { asin: row.asin, date: snapshotDate, metrics: row },
        })
      )
    )
  } else {
    // 其他报表：覆盖最新版本
    await db.contextFile.upsert({
      where:  { fileType },
      update: { fileName: file.name, snapshotDate, parsedRows: rows },
      create: { fileType, fileName: file.name, snapshotDate, parsedRows: rows },
    })
  }

  // 4. 保存原始文件到 context/ 目录
  const destPath = path.join(process.cwd(), "context", file.name)
  await fs.writeFile(destPath, buffer)

  // 5. 运行告警引擎，持久化结果（确保 Chat 的 get_alerts() 始终有数据）
  //    告警引擎依赖：product + keyword_monitor + inventory + us_campaign_30d
  //    仅当本次上传的 fileType 是上述依赖文件之一时才触发
  await runAndPersistAlerts(fileType)    // → Alert 表（upsert by asin + rule + date）

  return Response.json({ fileType, snapshotDate, rowCount: rows.length })
}
```

### 1.3 Parser 结构（以搜索词重构为例）

Nordhive 导出文件有固定的多行表头结构：

```
行1：筛选条件行（账号 / 站点 / 时间范围 / 视图...）
行2：筛选值（如 "2026-03-08 - 2026-04-06"）
行3：空行
行4：列名行（ASIN / 搜索词 / 匹配类型...）
行5+：数据行
```

```ts
// lib/parsers/searchTerms.ts
import * as XLSX from "xlsx"

export function parseSearchTerms(buffer: Buffer): { rows: SearchTermRow[], snapshotDate: string } {
  const wb    = XLSX.read(buffer)
  const ws    = wb.Sheets[wb.SheetNames[0]]
  const raw   = XLSX.utils.sheet_to_json(ws, { header: 1 }) as any[][]

  // 提取报表日期（D2 单元格）
  const dateRange   = raw[1][3] as string  // "2026-03-08 - 2026-04-06"
  const snapshotDate = dateRange.split(" - ")[1]  // 取结束日期

  // 第4行是列名，第5行起是数据
  const headers = raw[3] as string[]
  const dataRows = raw.slice(4).filter(r => r[0] && String(r[0]).startsWith("B0"))

  const rows: SearchTermRow[] = dataRows.map(r => ({
    asin:            r[headers.indexOf("ASIN")],
    search_term:     r[headers.indexOf("搜索词")],
    match_type:      r[headers.indexOf("匹配类型")],
    impressions:     Number(r[headers.indexOf("曝光量")]) || 0,
    clicks:          Number(r[headers.indexOf("点击量")]) || 0,
    spend:           Number(r[headers.indexOf("花费")]) || 0,
    ad_orders:       Number(r[headers.indexOf("广告订单量")]) || 0,
    ad_sales:        Number(r[headers.indexOf("广告销售额")]) || 0,
    ctr:             Number(r[headers.indexOf("CTR")]) || 0,
    cpc:             Number(r[headers.indexOf("CPC")]) || 0,
    acos:            Number(r[headers.indexOf("ACoS")]) || null,
    conversion_rate: Number(r[headers.indexOf("转化率")]) || 0,
    campaign:        r[headers.indexOf("广告活动")],
    placement_type:  r[headers.indexOf("投放类型")],
  }))

  return { rows, snapshotDate }
}
```

## 三、规则引擎结构

```ts
// lib/rules/sop/index.ts

interface SopInput {
  searchTerms:          SearchTermRow[]
  campaignWithBudget:   CampaignRow[]   // campaign_3m JOIN us_campaign_30d
  placement:            PlacementRow[]
  categoryAsins:        string[]
  config:               typeof getParam
}

export function runSopRules(input: SopInput): ActionItem[] {
  return [
    ...applyP0Rules(input),
    ...applyP1Rules(input),
    ...applyP2Rules(input),
    // ...applyP3Rules(input),  // P3 暂不启用：函数在 p3.ts 中完整实现，需要时取消注释
    ...detectOverlap(input),    // 跨 ASIN 词重叠
    ...detectZombies(input),    // 僵尸广告组
  ].sort(byPriority)
}

// lib/rules/sop/p0.ts  —  示例规则
export function applyP0Rules(input: SopInput): ActionItem[] {
  const items: ActionItem[] = []
  const clicksThreshold = input.config("P0_A_clicks_threshold")  // 默认 15
  const spendThreshold  = input.config("P0_B_spend_threshold")   // 默认 20

  for (const row of input.searchTerms) {
    // P0-A: 高点击 0 转化
    if (row.clicks >= clicksThreshold && row.ad_orders === 0) {
      items.push({
        priority: "P0", rule: "P0-A", asin: row.asin,
        term: row.search_term, match: row.match_type,
        clicks: row.clicks, orders: 0, spend: row.spend, acos: null,
        action: "暂停 + 否定",
        reason: `${row.clicks} 次点击，0 成交，花费 $${row.spend.toFixed(1)}`,
      })
    }
    // P0-B: 高花费 0 转化
    if (row.spend > spendThreshold && row.ad_orders === 0) {
      items.push({ priority: "P0", rule: "P0-B", ... })
    }
  }
  return items
}
```

## 四、数据新鲜度判断

Context 面板中文件卡片的颜色状态由以下逻辑决定：

```ts
// lib/config.ts 中定义新鲜度阈值（天）
const FRESHNESS = {
  product:         { fresh: 1, ok: 2 },   // 每日上传，>2 天变 stale
  keyword_monitor: { fresh: 2, ok: 7 },
  search_terms:    { fresh: 5, ok: 14 },
  campaign_3m:     { fresh: 10, ok: 45 },
  us_campaign_30d: { fresh: 5, ok: 10 },
  placement_us_30d:{ fresh: 5, ok: 10 },
  inventory:       { fresh: 3, ok: 7  },
  cost_mgmt:       { fresh: 30, ok: 60 },
  aba_search:      { fresh: 30, ok: 90 },
}

function getFreshness(fileType: string, uploadDate: Date): "fresh" | "ok" | "stale" {
  const daysAgo = (Date.now() - uploadDate.getTime()) / 86400000
  const thresholds = FRESHNESS[fileType]
  if (daysAgo <= thresholds.fresh) return "fresh"
  if (daysAgo <= thresholds.ok)    return "ok"
  return "stale"
}
```
